TRICK 1 LIMIT X OFFSET Y

limit 是限制 SQL 执行的结果，一般是后面紧接一个正整数作为参数，但还可以添加一个 OFFSET (游标),例如：

1

limit 4 offset 3 --意味从行3取4行

TRICK 2 使用限制表名

在SQL语句中，如果有必要，可以指定执行SQL的数据库和表：例如，操作 a 数据库中的 b 表的 c 列

1

select b.c from a.b

TRICK 3 ORDER BY

应该养成排序的好习惯，order by 后加列名，检索出的数据按字典序排序。 可以指定多个列名，执行时会先从第一个列名排序，而后在排序好的结果中，再按第二个列名排序 可以使用 desc 关键字来指定从 Z-A 的排序

TRICK 4 检索最值

使用

1

order by x limit 1

可以检索出最大和最小值

TRICK 5 正则表达式

REGXP 比 like 模糊查询要强大得多，REGXP 使用 ^$ 作为匹配字串得首尾占位符，.作为任意通配符， \ 作为转义字符（REG和MySQL各一个） 匹配多个实例得方法是使用重复元字符，它们有：*/+/?/{n}/{n,}{n,m}

TRICK 6 创建计算字段

计算字段是指，通过SQL得一些格式化函数，将多个字段拼接或计算成一个业务所需要的字段。 例如：

1
2
3

-- Concat 拼接函数
select Concat(vend_name,'(', vend_country, ')') from vendors order by vend_name;
-- | ACME (USA) |

上面输出的字段名如果不换一个字段名是很傻的， 使用 别名(alias) 可以使新的字段名变得简洁:

1

select Concat(vend_name,'(', vend_country, ')') as vend_tilte from vendors;

除此之外，使用 Rtrim/Ltrim 或 Trim 函数可以去除字段右/左或两边的空格，使用四则运算可以得到一个新的计算字段.

TRICK 8 聚集函数汇总数据

在使用上述函数的时候，可以通过where字句调整需要汇总的数据 例如：

select MAX(price) as max_price from products where class = 'ITBook'

汇总数据的时候，默认是汇总所有的数据（默认使用关键字 ALL ），包括重复值，使用 DISTINCT 关键字可以排除重复值，对于重复的值只计算汇总一次，例如：

1

select MAX(DISTINCT price) as max_prince from products where class = 'ITBook'

DISTINCT 在 count(*) 函数中使用没有意义，使用将报错

在 MIN()/MAX() 函数中可以使用，但最大最小值只有一个，使用也没有意义

TRICK 9 使用分组

1

select id,count(*) as num from products group by id;

注意，如果使用 where 字句，必须让 where 语句放在 group by 字句之前 过滤分组时，可以使用 HAVING 字句

where 和 having 字句不同之处在于，where 是对行进行过滤（分组前），而 having 时对查询结果（分组后）进行筛选过滤 例如，要筛选价格超过十块，销量大于2的商品的销量，并且按商品 id 进行分组：

1
2
3
4
5
6


select id,count(*) as num 
from products
where price >= 10
group by id
having count(*)>=2;

group by 和 order by分组和顺序可以结合使用，区别是，group by 只能对选择的列使用，换句话说，后边参数只能是 select 的列或列表达式 order by 可以对非选择的列使用，来看一个两者结合使用的例子

1
2
3
4
5

select id,count(*) as num 
from products
where price >= 10
group by id
having count(*)>=2

TRICK 10 使用子查询

看两个例子。

  • 有三张表，一张是用户表，存放 uid 和 name ；一张物品表，存放 item_id 和 item_name ；一张购物表，存放 order_id、uid 和 item_id ,现在要查询购买了item_name 为 TNT2 的用户的 name和phone，SQL语句要怎么写呢？

思路：先在item表中检索’TNT’的item_id,再在order表中检索出出购买了’TNT’的uid，再通过uid检索name和phone

1
2
3
4

select name, phone 
from user where uid in(select uid 
                      from order where item_id in(select item_id 
                                                  from item where item_name = 'TNT'));
  • 统计 order 表中每个用户的订单总数，该怎么写？

思路：从 user 表中检索用户列表；对于每个用户，统计其在 orders 表中的订单数目，使用 count() 计算属性

我们先来看看单个用户的 SQL 怎么写：

1
2
3

select count(*) as order_num
from order
where uid = 1001

1
2
3
4
5

select name,
      (select count(*) from order 
      where order.id=user.uid) as order_num
from user
order by name;

这种涉及到外部查询的子查询也叫相关子查询（correlated subquery`）

TRICK 11 使用联结表

如果数据存储在多个表中，怎样用单条 select 语句检索出数据？ 答案是使用联结表。 但应该注意，因为联结表会创造现实中不存在的“虚拟表”，后查询，因此，性能比子查询差。会多耗内存和计算资源。 能使用子查询的情况下推荐使用子查询。

等值联结也称为内部链接。

很多场景下，使用”外键”能够实现表与表之间的引用约束，但可能会增加性能消耗，因为使用外键约束会造成每次CURD时都会检索引用是否存在。创建时检测，删除时也检测外键约束。一般中大型的应用是不推荐使用外键的。

trick 10 中查询购买了TNT的用户信息，也能使用联结实现：

1
2
3
4
5

select user.name,user.phone 
from user,item,order
where user.uid = order.uid and
item.item_id = order.item_id and
item.item_name = 'TNT';

TRICK 12 高级联结

1.表别名实现自联结

自联结，顾名思义就是一张表，自己和自己连，听起来似乎很扯淡。但思考以下场景： 现在仓库里某一批名为 DTNTR 的货物有问题，你想找出该批货物供应商供应的其他货物名，但你还不知道该物品的供应商 ID ，应该如何做？ 数据库里只有一张表，货物供应表，该表有：物品 ID,物品名，供应商 ID； 你可能第一想到的方法是使用子查询，先查找出名为 DTNTR 的物品的供应商ID；然后利用供应商ID查找出所有该供应商提供的物品。 于是你写了子查询：

1
2
3
4
5

select prod_id, prod_name
from products
where vend_id = (select vent_id
                  from products
                  where prod_name = 'DTNTR';)

这是一种方法，看起来不错，但是嵌套了一层查询，看起来不简洁，让我们使用自联结实现相同的功能，看看是否简洁一些：

1
2
3
4

select p1.prod_id, p1.prod_name
from products as p1, products as p2
where p1.vend_id = p2.vend_id
and p2.prod_name =  'DTNTR';

自联结比查询会快一些

2.自然联结

所谓自然联结，就是联结时一张表表使用通配符*（或其他你想要的列）其他表明确这张表没有出现过的列，排除多次出现

3.外部联结

还是考虑顾客购买商品的场景：

你希望对每个客户下了多少订单进行计数，包括那些从未下单的客户； 列出所有产品以及订单数量，包括没有人订购的产品； 计算平均销售规模，包括那些从未下单的客户；

1
2
3

select customers.cust_id, orders.order_id
from customers LEFT JOIN orders 
on customers.cust_id = orders.cust_id;

该SQL会查询出所有顾客的订单，包括没下单的顾客.

当我们需要查出所有顾客的订单数量时,

1
2
3
4
5
6

select customers.cust_id,
      customers.name,
      count(orders.cust_id) as num_od
from customers LEFT JOIN orders 
on customers.cust_id = orders.cust_id
group by customers.cust_id;

使用内连接查出下了订单的顾客的订单

1
2
3
4
5
6

select customers.cust_id,
      customers.name,
      count(orders.cust_id) as num_od
from customers INNER JOIN orders 
on customers.cust_id = orders.cust_id
group by customers.cust_id;

区别是什么呢？ 内连接不会筛选记录为空的行，左外连接会把没有下订单的客户的订单数（0）也能筛选出来.
