他山之石-SMART 目标管理法则

如果你不知道去往何处，那么你必须多加小心，因为你很可能无法到达那里。

——Yogi Berra , 前美国纽约扬基棒球球队球手

SMART 是为目标实现提供了一个靠谱的方法论，在咨询、个人成长领域被广泛使用。SMART 代表：

  • Specific: 具体的 —— 把要实现的目标具体化

例如，“我想学习 Kubernetes“ 是不够的，应该更具体化，例如：”我想用 Kubernetes 搭建一个容器平台，并利用 Jenkins 和 Gitlab 构建 DevOps CI/CD 流水线”。

  • Measurable: 可度量的 ——完成该目标需要多长时间？

用确切的数字去度量实现目标任务需要付出的成本。如果无法度量自己的目标任务，那么它很可能还不够具体。度量你的任务，但是要采取增量的方法。

你不必看清你去往何处（心里清楚就好），不必看清目的地和沿途的一切，只需要看清前面的一两米即可。

  • Achievable: 可实现的

一个你无法达到的目标不是目标，只是一种疯狂的、吸食灵魂的自我挫败。

  • Relevant：相关的

目标真的与你有关吗？你对此有热情吗？是在你掌控之下的事情吗？如果不是，这个目标就是不相关的。

  • Time-boxed：时间可控的

虽然是最后一个特性，却可能是最重要的一个。你需要为目标设定一个最后期限，没有期限，目标会逐步衰退，永远被每天紧急的事情所排挤，这样它永远都不会实现。

对于任何目标（减肥、炒老板鱿鱼、征服世界等），你都需要制定一个计划，列出一系列帮助你实现目标的任务(objective)。每个目标都应该具有 SMART 特性。

使用 Minikube 部署 Kubernetes 集群

一、创建集群 1.1 启动 minikube minikube verison minikube start 如果使用本地环境，需要启用 hyper-v 并配置虚拟交换机, 由于🧱存在，不建议在本地使用minikube， 如果想在本地环境学习 Kubernetes，可以使用阿里云提供的 k8s 镜像服务 1.2 查看集群状态 # 版本信息 kubectl version # 集群状态 kubectl cluster-info # 节点信息 kubectl get nodes 二、部署应用 kubectl 使用 k8s API 和集群交互，通过它我们可以管理我们的集群。 三、探索应用 3.1认识 k8s pods 当我们发起一个部署任务的时候，k8s 会创建一个 Pod 为应用实例提供一个运行环境。Pod 是 k8s 里的一个抽象概念，可以理解为一组或单个应用容器共享资源的集合。这些资源可能会包括： 共享存储卷 网络，每个 Pod 在集群里都有一个唯一的 IP 地址 每个容器的详细信息，包括容器镜像的版本信息（镜像地址、版本号）或端口映射关系 一个 Pod 相当于应用的逻辑主机，可以同时运行多个关系比较紧密的多个应用容器。Pods 是 k8s 平台里的不可分割的最小单元，每个 Pod 会和调度它的 Node 绑定。当然，假如 Node 节点故障了，对应的 Pods 会被其他健康的 Nodes 调度，我们可以通过对 Node 设置标签和选择器来决定哪些 Nodes 可以调度该 Pods。

goroutine 并发控制

一、 sync.WaitGroup 控制 goroutine 协调完成

func main() {
	var wg sync.WaitGroup
	wg.Add(2)
	go func() {
		time.Sleep(2*time.Second)
		fmt.Println("1号完成")
		wg.Done()
	}()
	go func() {
		time.Sleep(2*time.Second)
		fmt.Println("2号完成")
		wg.Done()
	}()
	wg.Wait()
	fmt.Println("好了，大家都干完了，放工")
}

使用 sync.WaitGroup 能够优雅地实现等待多个 goroutine 完成。

CentOS离线安装MySQL

如果你的 CentOS 运行在内网环境访问不了公网，内网环境由没有配置相关的源，那么可以考虑离线安装。

查看 CentOS 版本

# cat /etc/centos-release

下载rpm

repo.mysql.com/yum

选择对应的版本下载：

  • libs
  • common
  • client
  • server

安装

# yum localinstall mysql-xxxx.rpm

注意需要client、server依赖common、libs，因此四个包都需要安装。

题解 | Leetcode 76 - Minimum Window Substring 最小覆盖子串

审题

给定一个字符串 S 和一个字符串 T，请在 S 中找出包含 T 所有字母的最小子串。

示例：

输入: S = "ADOBECODEBANC", T = "ABC"
输出: "BANC"

说明：

如果 S 中不存这样的子串，则返回空字符串 ““。 如果 S 中存在这样的子串，我们保证它是唯一的答案。
